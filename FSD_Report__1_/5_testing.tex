\chapter{TESTING}
Testing is a fundamental phase in the development of the Route Optimizer, ensuring that the optimization algorithms are accurate, API integrations are resilient, and the user interface remains responsive and intuitive. Comprehensive testing helps identify edge cases in geographical calculations, potential failures in external data fetching, and security vulnerabilities in the authentication flow. This chapter describes the testing strategy employed across multiple levels: unit testing, integration testing, system testing, performance testing, and user acceptance testing.

\section{Testing Strategy}
The testing approach for Route Optimizer follows a layered pyramid model:
\begin{enumerate}
    \item \textbf{Unit Tests:} Validate individual utility functions and component logic in isolation.
    \item \textbf{Integration Tests:} Verify correct interactions between frontend, backend, and external APIs.
    \item \textbf{System Tests:} Evaluate end-to-end user journeys from registration through to itinerary saving.
    \item \textbf{Performance Tests:} Measure response times and ensure the application meets responsiveness benchmarks.
    \item \textbf{User Acceptance Tests (UAT):} Confirm that the application meets the user's expectations in real-world scenarios.
\end{enumerate}
